# AI놀자 홈페이지 사용 설명서

## 사이트 URL
- **프론트엔드**: https://ainolza-web.stressoutcompany.workers.dev
- **관리자**: https://ainolza-web.stressoutcompany.workers.dev/manager

---

## 1. 전체 구조

```
프론트엔드 (방문자용)          관리자 (콘텐츠 관리)
─────────────────────        ─────────────────────
/              홈페이지        /manager           대시보드
/store         스토어          /manager/products  상품 관리
/community     커뮤니티        /manager/posts     게시판 관리
/programs      프로그램        /manager/programs  프로그램 관리
/p/{slug}      커스텀 페이지   /manager/design    페이지 빌더
```

---

## 2. 홈페이지 만들기 (가장 먼저 할 일)

1. `/manager` 접속
2. 왼쪽 사이드바 > **페이지** 클릭
3. **"새 페이지"** 버튼 클릭
4. 제목: 아무거나 (예: "홈페이지")
5. **슬러그: `home`** (반드시 home으로 설정!)
6. 저장 후 에디터에서 드래그앤드롭으로 디자인
7. 상태를 **Published**로 변경 후 저장
8. `/` 접속하면 만든 홈페이지가 표시됨

> **핵심**: 슬러그가 `home`인 Published 페이지 = 홈페이지

---

## 3. 네비게이션 메뉴 탭 추가하기

### 현재 메뉴
헤더에 표시되는 메뉴는 `src/components/layout/header.tsx` 파일에서 관리합니다.

```typescript
// 이 배열을 수정하면 상단 메뉴가 바뀝니다
const navItems = [
    { name: "커뮤니티", href: "/community" },
    { name: "스토어", href: "/store" },
    { name: "프로그램", href: "/programs" },
];
```

### 메뉴 추가 방법

**방법 1: 기존 섹션 추가** (페이지가 이미 있는 경우)
```typescript
const navItems = [
    { name: "커뮤니티", href: "/community" },
    { name: "스토어", href: "/store" },
    { name: "프로그램", href: "/programs" },
    { name: "소개", href: "/p/about" },        // 페이지 빌더로 만든 페이지
    { name: "문의", href: "/p/contact" },       // 페이지 빌더로 만든 페이지
];
```

**방법 2: 페이지 빌더로 커스텀 페이지 만들기**
1. `/manager/design` > 새 페이지 만들기
2. 슬러그를 `about`, `contact` 등으로 설정
3. 에디터에서 내용 디자인
4. Published로 저장
5. header.tsx에 `{ name: "소개", href: "/p/about" }` 추가
6. git push하면 자동 배포

> **참고**: 프론트엔드 코드(header.tsx) 수정 후에는 `git push`가 필요합니다.
> 관리자에서 만드는 콘텐츠(상품, 게시글 등)는 push 없이 즉시 반영됩니다.

## 4. 상품 관리

### 상품 등록
1. `/manager/products` > **"새 상품 등록"**
2. 필드 입력:
   - **제목** (필수): 상품 이름
   - **슬러그**: URL 경로 (자동 생성됨, 예: `ai-course`)
   - **가격** (필수): 원(₩) 단위
   - **설명**: 짧은 소개
   - **상세 내용**: 긴 설명
   - **카테고리**: 분류 선택
   - **상태**: Draft(임시저장) / Published(공개)
   - **추천 상품**: 체크하면 목록 상단에 표시
3. 저장

### 프론트엔드 표시
- `/store` → Published 상품 목록 (그리드)
- `/store/{slug}` → 개별 상품 상세 페이지

---

## 5. 커뮤니티(게시판) 관리

### 게시글 작성
1. `/manager/posts` > **"새 게시글"**
2. 필드 입력:
   - **제목** (필수)
   - **내용** (필수)
   - **작성자**: 기본값 "관리자"
   - **카테고리**: 자유 / 질문 / 공유 / 공지
   - **상태**: 임시저장 / 게시
   - **상단 고정**: 체크하면 목록 최상단에 표시
3. 저장

### 프론트엔드 표시
- `/community` → 게시글 목록 (카테고리 필터 있음)
- `/community/{id}` → 게시글 상세 + 댓글

### 댓글 관리
- 방문자가 게시글 하단에서 댓글 작성 가능
- 댓글은 승인(approved) 상태만 표시됨

---

## 6. 프로그램 관리

### 프로그램 등록
1. `/manager/programs` > **"새 프로그램"**
2. 필드 입력:
   - **제목** (필수)
   - **슬러그**: URL 경로 (자동 생성)
   - **설명**: 짧은 소개
   - **버전**: 예: v1.0.0
   - **다운로드 URL**: 다운로드 링크
   - **플랫폼**: Windows / Mac / Linux / Web / 전체
   - **카테고리**: 유틸리티 / AI도구 / 플러그인 / 기타
   - **상태**: 임시저장 / 공개
   - **추천**: 체크하면 목록 상단에 표시
3. 저장

### 프론트엔드 표시
- `/programs` → 프로그램 목록 (그리드)
- `/programs/{slug}` → 상세 페이지 + 다운로드 버튼

---

## 7. 페이지 빌더 (GrapeJS)

### 사용법
1. `/manager/design` > **"새 페이지"**
2. 제목과 슬러그 입력
3. 에디터에서:
   - 왼쪽 패널: 블록/컴포넌트 드래그
   - 중앙: 캔버스 (실시간 편집)
   - 오른쪽: 스타일/속성 조정
4. Published로 변경 후 저장

### 만들 수 있는 페이지
| 슬러그 | 용도 | 접근 URL |
|--------|------|----------|
| `home` | 홈페이지 | `/` |
| `about` | 소개 페이지 | `/p/about` |
| `contact` | 문의 페이지 | `/p/contact` |
| 아무거나 | 커스텀 페이지 | `/p/{slug}` |

## 9. 콘텐츠 vs 코드 변경

| 작업 | 방법 | 배포 필요? |
|------|------|-----------|
| 상품/게시글/프로그램 등록 | `/manager`에서 관리 | **아니오** (즉시 반영) |
| 페이지 빌더로 페이지 만들기 | `/manager/design`에서 관리 | **아니오** (즉시 반영) |
| 네비게이션 메뉴 수정 | `header.tsx` 코드 수정 | **예** (git push) |
| 새 섹션/기능 추가 | 코드 개발 | **예** (git push) |

> git push하면 GitHub Actions가 자동으로 빌드 + Cloudflare 배포합니다 (약 2분 소요).

## 10. 자주 묻는 질문

**Q: 이미지는 어떻게 올리나요?**
A: 현재 관리자 폼에 이미지 업로드 UI가 없습니다. Payload CMS 관리 패널(`/admin`)에서 Media 컬렉션으로 업로드하거나, 외부 이미지 URL을 사용하세요.

**Q: 커스텀 도메인은 어떻게 연결하나요?**
A: Cloudflare 대시보드 > Workers & Pages > ainolza-web > Custom Domains에서 설정합니다.

**Q: 데이터 백업은?**
A: Cloudflare D1 데이터베이스에 저장됩니다. Cloudflare 대시보드에서 D1 > ainolzadb에서 확인/백업 가능합니다.
